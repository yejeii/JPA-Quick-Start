
다대다 연관관계 매핑( p.317 )

1. ERD 기준으로 다대다
    - 주문 테이블(S_ORD), 상품 테이블(S_PRODUCT)
    - 상기의 두 테이블을 다대다로 관계를 맺어주기 위해 관계설정용 테이블이 필요함(S_ITEM 테이블)
    - 관계설정용 테이블은 정규화를 통해 중복되는 데이터를 없애기 위해 만들어진 것
    - 관계를 맺어주기 위해 S_ITEM 칼럼은 S_ORD, S_PRODUCT 의 PK 를 FK 로 사용하고,
      이 두 개의 FK 가 복합키로 사용됨

2. 상기의 다대다 테이블 관계를 엔티티 기준으로 설계
    2.1 관계 설정 방법
        2.2.1 엔티티 두 개로 관계 설정
              S_ORD 에 대한 엔티티와 S_PRODUCT 엔티티가 필요하고,
              S_ORD 와 S_PRODUCT 의 관계 설정용 엔티티는 사용하지 않음

        2.2.2 식별관계
              S_ORD 에 대한 엔티티와 S_PRODUCT 엔티티의 각각의 식별자 변수를
              S_ITEM 의 관계 설정용 FK 로 사용하여 복합키로 사용

        2.2.3 비식별관계('비' 가 식별관계의 반대 의미를 지칭하진 않음)
              S_ORD 에 대한 엔티티와 S_PRODUCT 엔티티의 각각의 식별자 변수를
              S_ITEM 의 관계 설정용 FK 로 사용하고,
              S_ITEM 의 PK 는 별도의 식별자 변수로 사용

    2.2 다대다 단방향( 식별관계 ) - 실습용
