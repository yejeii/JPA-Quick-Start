
JPA ( Java Persistence API )

식별자

1.  JPA 는 엔티티 클래스로부터 생성된 엔티티를 기반으로 데이터의 영속성을 관리한다.
    엔티티 클래스를 DB 테이블과 매핑하는 것이 중요.
    ( DB 설계가 기반이 됨 )

2.  JPA 는 엔티티를 관리할 때, 엔티티에 할당된 유일한 식별자 값을 사용한다.
    따라서, 식별자 변수가 지정되지 않은 엔티티는 사용할 수 없다.
    식별자 변수(@ID 멤버)에 값이 할당되지 않은 엔티티는 JPA 의 관리 대상이 아니다.

3.  JPA 는 식별자 변수에 유일한 값을 자동으로 할당하기 위해 4가지 전략을 제공한다.
    IDENTITY, SEQUENCE, TABLE, AUTO 
    이와 함께, 복합키를 식별자 클래스 기반으로 사용할 수 있도록 지원하고 있다.
